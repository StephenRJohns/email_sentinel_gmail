PART 1 — CLOUD CONSOLE NAVIGATION
================================================================================

Most of this work happens in Google Cloud Console, not in this repo.

STEP-BY-STEP:

  1. Go to https://console.cloud.google.com
  2. Top bar → select the Cloud project that is linked to your Apps Script
     project. If you don't know which one, run `clasp open` locally, then in
     the Apps Script editor go to Project Settings → the "Google Cloud
     Platform (GCP) Project" section — that's the one.
  3. Left hamburger menu → APIs & Services → Library
  4. Search "Google Workspace Marketplace SDK" → Enable (if not enabled)
  5. Left hamburger menu → APIs & Services → Enabled APIs & services →
     click "Google Workspace Marketplace SDK"
  6. You are now at the Marketplace SDK page. You will use three tabs here:
       • OVERVIEW          — status
       • APP CONFIGURATION — where you paste the descriptions, URLs, assets
       • STORE LISTING     — where you preview the listing before submitting

  7. Click APP CONFIGURATION tab. Fill out:
       • App Name
       • App integration (add-on details)
       • OAuth scopes (must match appsscript.json)
       • Developer details (your name/email)
       • Terms of Service URL
       • Privacy Policy URL
       • Visual assets (icon, banner, screenshots)
       • Detailed description
  8. Click STORE LISTING tab:
       • Short description
       • Category
       • Regions to publish
       • Languages
  9. Save (do NOT click Publish yet — do that only when OAuth verification
     is done and everything below is complete).

3. DETAILED DESCRIPTION
--------------------------------------------------------------------------------
   Max 16,000 characters. Appears on the listing page below screenshots.
   This is your biggest SEO surface — include natural keywords.

   emAIl Sentinel™ is smart email management and alerting for Gmail.
   Describe what to watch for in plain English — AI evaluates every new
   message against your rules and alerts you through the channels you
   choose. No regex, no code, no extra accounts.

   WHY IT'S DIFFERENT
   • Plain-English rules. "Any email from the bank confirming a wire
     transfer over $10,000." Done.
   • Evaluated by Google Gemini. Real semantic understanding, not keyword
     matching.
   • Multi-channel alerts per rule. One email can text you, create a
     calendar event, and log to a spreadsheet simultaneously.
   • Runs entirely inside your own Google account. No third-party backend,
     no extra account to create, your API key stays in your UserProperties.

   ALERT CHANNELS (mix and match per rule)
   • SMS — bring your own provider. YOU select and configure the SMS
     provider; emAIl Sentinel dispatches messages on your behalf using the
     credentials you supply. Quick-start presets make setup trivial for
     Textbelt, Telnyx, Plivo, Twilio, ClickSend, and Vonage. A generic
     webhook handles every other provider. emAIl Sentinel is not itself an
     SMS provider and does not resell messaging.
   • Google Chat — post to any Chat Space via webhook (Pro)
   • Google Calendar — creates an event with phone notifications
   • Google Sheets — appends a row for audit trails and searchable history
   • Google Tasks — creates a task in your Gmail sidebar
   • External integrations — Microsoft Teams, Asana (REST or MCP V2),
     any custom Model Context Protocol endpoint, or any HTTPS webhook
     (Slack incoming, Discord, n8n / Zapier, custom internal APIs) (Pro)

   HOW IT WORKS
   A background trigger checks Gmail on the interval you choose. For each
   new message in a watched label, Gemini reads the sender, subject, body,
   and attachment names and decides whether it matches your rule. If yes,
   it formats an alert message and fires every channel you configured.
   Messages that don't match are ignored entirely — no noise.

   PRICING
   • Free — the add-on is fully featured: unlimited rules, every alert
     channel (SMS, Google Chat, Calendar, Sheets, Tasks, Docs, MCP,
     webhooks), AI-assisted rule writing, and full log history, at no cost.
   • Scans run at Google's 1-hour trigger floor; "Scan email now" runs an
     immediate scan any time.
   • Always-on, real-time monitoring is a separate self-hosted product,
     emAIl Sentinel Pro ($15/month, $150/year, or $149 lifetime).

   IMPORTANT — POLLING PLATFORM LIMIT
   Google enforces a 1-hour minimum on time-based triggers in every
   Workspace add-on. This is a Google platform constraint, not a pricing
   choice; we cannot poll faster than this. The add-on is free with no
   tiers, so this floor applies to everyone; "Scan email now" runs an
   immediate scan any time.

   EXAMPLE RULES

   • "Email from the bank confirming a wire transfer, ACH payment, or

   PRIVACY
   Everything important stays in your Google account. Rules, settings,

12. OAUTH VERIFICATION
--------------------------------------------------------------------------------
    Because gmail.readonly is a restricted/sensitive scope, you must
    submit for OAuth verification before Marketplace approval.

    In Cloud Console → APIs & Services → OAuth consent screen →
    Verification status → Prepare for verification.

    Google requires:
      • A video walkthrough showing how the add-on uses each scope
        (your OAuth verification video — see docs/oauth_verification/)
      • A written justification for each scope (below)
      • A link to your privacy policy

    SMS COMPLIANCE NOTE
    If users will be testing SMS providers (especially Twilio), they must
    complete A2P 10DLC registration with their provider. Twilio requires
    campaign registration in the console (A2P 10DLC > Register). Without
    this, test SMS will fail with error 30054 (Unregistered Number).
    The app provides guidance in error messages.

    SCOPE JUSTIFICATIONS — paste these into the verification form:

      gmail.addons.execute
        Required to run as a Gmail add-on. This scope is granted
        implicitly by installing any Gmail add-on.

      gmail.readonly
        Required to read message metadata (sender, subject, received date,
        attachment filenames) and up to the first 2,000 characters of the
        plain-text body, for rule evaluation by the Google Gemini API.
        The add-on only reads messages in Gmail labels the user explicitly
        configures. No messages are modified, sent, or deleted.

      calendar
        Required to create calendar events as alerts when the user enables
        the Google Calendar alert channel on a rule. CalendarApp requires
        the full calendar scope; the narrower calendar.events scope is
        only compatible with the advanced Calendar service, which is not
        used here.

      spreadsheets
        Required to log alert data to Google Sheets when the user enables
        the Google Sheets alert channel on a rule.

      tasks
        Required to create Google Tasks as alerts when the user enables
        the Google Tasks alert channel on a rule.

      script.external_request
        Required to call the Google Gemini API (generativelanguage.
        googleapis.com) for rule evaluation and alert formatting, and to
        call user-selected third-party SMS providers (Textbelt, Telnyx,
        Plivo, Twilio, ClickSend, Vonage) and MCP server endpoints
        (Slack, Microsoft 365 / Teams, Asana, or user-configured
        custom endpoints) when the user enables those alert channels.

      script.scriptapp
        Required to create and manage time-driven triggers that schedule
        background email checks at the user-configured polling interval.
